## 2. 全局约束与技术债（PRD §3、§4、§9、§10）

| PRD / Step2 要求 | 当前实现 | 差距说明 |
| ---------------- | -------- | -------- |
| **§3.1 JS 运行时兼容 ES5**；构建产物降级 + 按需 polyfill | Vite 默认输出现代语法；未接入 `@vitejs/plugin-legacy` / Babel `preset-env` 等到 ES5 | **未满足**：目标机（Chrome 69 / iOS 12）上可能存在语法/API 风险，需单独验收与构建改造。 |
| **§3.2 PostCSS 适配**；PC/Pad/Phone 比例一致 | 已配置 `postcss-px-to-viewport`（375） | **基本满足**；Tailwind 大量 `rem`/任意值与插件协同需在真机再扫一遍。 |
| **§3.3 viewport** | `index.html` 已含 `maximum-scale=1.0, user-scalable=no` | **已满足**。 |
| **§3.4 拖拽**：Touch + Mouse；禁止 HTML5 DnD | 麦克风等使用 Pointer；未实现 **P4 横向滑动手势翻页**（Step2 列为可选） | **部分差距**：翻页仍为按钮；若 PRD 冻结要手势，需补 `touchmove` 阈值逻辑。 |
| **§3.5 一屏无滚动**；Pad 横屏可整体略缩放 | 多数页一屏；**未实现根层 `transform: scale` / 根字号做 Pad 整体缩放**（Step2 §2.2） | **部分差距**：大屏/Pad 主要为 `max-w-[375px]` 居中，非「整体略缩放」方案。 |
| **§3.6 禁止第三方业务 H5** | 无外链 iframe/跳转 | **已满足**。 |
| **§3.7 字体苹方/雅黑** | `index.css` 等已设 `PingFang SC`、`Microsoft YaHei` | **已满足**方向；未做全站组件级审计。 |
| **§4 技术栈** Howler 等 | 朗读用原生 `Audio` + Web Audio 蜂鸣；未引入 Howler | **可选差距**：非强制，属实现选型差异。 |
| **§9 图片 JPG/PNG**；音频 MP3 128kbps 放 `public` | 相册入口支持选图；绘本页、朗读音频 **多为占位或缺失文件回退**（计时/蜂鸣） | **素材与格式**：逻辑支持路径；**未批量提供** PRD 所列封面/分页图与 MP3。详见 [**预设素材清单**](./预设素材清单.md)。 |
| **§9 WebP 优化** | 未系统性提供 WebP 双轨 | **可选差距**。 |
| **§10 转场、按钮缩小+震动、识别成功绿闪等** | 局部有 `active`、震动在扫码成功路径；**无统一「页面转场 / 弹性缓动」体系** | **体验差距**：属 polish，非阻塞流程。 |

## 6. 建议补齐优先级（供排期）

1. **高**（合规 / 验收）：**ES5 或目标浏览器构建策略**（§3.1）；真机抽测 **摄像头/录音/IndexedDB**。  
2. **高**（PRD 文案明确）：**§6.2 拍绘本** 多段音频 + 翻页/间隔模拟（可与 PRD 参数对齐为 3s）。  
3. **中**：**素材包**（§9）——分页 JPG/PNG、分页 MP3、BGM、扫码 beep；P4 真实插图替换占位。  
4. **中**：**Pad 整体缩放**（Step2）、P3/P4 **动效与 3D 视觉**向 §11 靠拢。  
5. **低**：§10 全局动效规范；P4 **横向滑动手势翻页**；P1 **内置 2～3 本演示书**。
