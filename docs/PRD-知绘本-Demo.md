## 2. 项目概述

### 2.1 产品定位

**知绘本**：面向 3–8 岁儿童的 AI 智能绘本朗读应用原型（纯前端）。通过手机摄像头识别纸质绘本（Demo 为模拟），系统朗读内容，并支持家长克隆音色入口（Demo 仅录音与本地保存，不训练）。

### 2.2 Demo 目标与边界

| 类别     | 说明 |
| -------- | ---- |
| 必须达成 | 完整主流程与辅助流程可交互；3D 卡通视觉风格；音频播放与关键动画；本地数据持久化。 |
| 模拟     | 识别：延迟 + 假数据；朗读：预录 MP3；音色克隆：仅录音保存，朗读仍用预设音频。 |
| 不做     | 后端 API、真实 OCR、真实 TTS、音色模型训练、外链第三方业务 H5。 |

### 2.3 成功标准（验证项）

- 交互流程顺畅，儿童与家长能理解主要入口与步骤。
- 视觉风格符合儿童审美（参考「猛兽派对」式 3D 卡通、高饱和、圆润）。
- 可为后续真实功能开发提供清晰原型参考。

## 3. 强制技术约束（H5）

以下约束 **高于** 一般技术选型偏好，实现方案须在设计与评审阶段逐项对照。

### 3.1 运行环境与语法兼容性

| 项 | 要求 |
| -- | ---- |
| 浏览器内核 | Chrome 内核 **69 及以上**。 |
| iOS | **12 及以上**。 |
| Android | **6 及以上**。 |
| JavaScript | **运行时行为须兼容 ES5 能力模型**：构建产物需降级（如 Babel 转译至 ES5），并对必要的语言特性缺口提供 **可控 polyfill**（按需引入，避免无谓体积）。源码可使用 TypeScript/React 编写，但交付 bundle 须满足目标环境。 |
| CSS | 避免仅新机支持的特性；复杂布局与动画优先 `transform` / `opacity` 等 GPU 友好属性。 |

**说明**：Chrome 69 / iOS 12 对部分现代 Web API 支持有限，具体 API（如 `MediaRecorder` 格式、`getUserMedia` 行为）需在开发阶段做能力检测与降级文案。

### 3.2 布局适配（PostCSS）

- **原则**：使用 **PostCSS 体系**做屏幕适配（例如基于设计稿宽度做 **vw / rem / 单位转换**），保证在 **PC / Pad / Phone** 下 **同一套视觉比例一致**（元素相对位置与缩放一致，而非简单拉伸错位）。
- **Step 2 已冻结**：设计基准宽度 **375px**；插件建议 **`postcss-px-to-viewport`**（详见 [`STEP2-交互与视觉规格.md`](./STEP2-交互与视觉规格.md) §1、§7）。

### 3.3 `viewport`

页面 HTML 须包含：

`width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no`

### 3.4 拖拽与手势（凡涉及拖拽的交互）

不使用 HTML5 **Drag and Drop API**。统一采用：

- **Touch**：`touchstart` / `touchmove` / `touchend`
- **Mouse**：`mousedown` / `mousemove` / `mouseup`（含桌面）

附加规则：

- 可拖拽或可拖动手势控制的元素：`touch-action: none`，避免拖拽时触发页面滚动。
- 拖拽过程中元素 **跟随指针/手指**，并保证 **`z-index` 足够高**，避免被遮挡。

**适用范围（实现时按页面落地）**：例如书架卡片拖动、横屏绘本翻页手势等——具体控件在 Step 2 交互稿中标明。

### 3.5 一屏展示与滚动

- **默认**：每个功能页应在 **一屏内完整展示**，尽量不出现整页滚动条。
- **例外**：**列表类文字信息**（如长文案、条款、音色列表过长）允许纵向滚动；需在交互稿中标注哪些区域可滚动。

### 3.6 第三方资源与外链

- **禁止**：业务代码中 **调用或跳转至第三方 H5 地址**（非本项目域名内的嵌套页、外链运营页等）。
- **允许**：npm 依赖经打包进自有静态资源；自有域名静态托管；本地 `public` 资源。

### 3.7 字体

优先使用系统字体栈：**苹方（PingFang SC）**、**微软雅黑（Microsoft YaHei）**，并附合理 fallback（如 `sans-serif`），避免依赖外网字体 CDN。

## 4. 技术栈（与约束对齐后的建议）

| 层级 | 建议 | 说明 |
| ---- | ---- | ---- |
| 框架 | React + TypeScript + Vite | 源码现代；构建链路须输出 ES5 兼容 bundle。 |
| 路由 | React Router v6 | — |
| 样式 | Tailwind CSS + PostCSS 适配插件 | Tailwind 仅开发效率；最终尺寸策略服从 §3.2。 |
| 动画 | **优先 CSS + 少量受控 JS** | Framer Motion 等库若引入，需评估转译后在旧环境的兼容性；必要时用 CSS 动画替代。 |
| 摄像头 | 自建 `<video>` + `getUserMedia` 或轻量封装 | 避免不可控外链；注意 HTTPS/localhost。 |
| 音频 | HTML5 `Audio` 或 Howler（若引入须打包） | Demo 预录 MP3，本地路径。 |
| 存储 | LocalStorage（书单元数据）+ IndexedDB（音色 Blob） | 可用 localforage 等封装，npm 打包。 |

**与原草案差异**：原稿中的库仍可选用，但必须 **bundle 内引用**，且满足 §3.1 / §3.6。

## 5. 功能模块与页面清单

| 序号 | 页面 | 核心职责 |
| ---- | ---- | -------- |
| P1 | 首页（书架） | 网格书架、空状态、底部 Tab（书架 / 开始读书 / 音色克隆）。 |
| P2 | 扫码 / 拍照页 | 摄像头预览、扫二维码 / 拍绘本模式切换、模拟识别。 |

## 6. 用户流程（摘要）

### 6.1 主流程：扫码识别 → 朗读

首页点击底部 **「开始读书」** → 进入 **扫码 / 拍照页（P2）**：

- **扫到带二维码的绘本（Demo 模拟）**：约 2s → **识别结果页（P3）** → 「马上开始读」→ **横屏提示** → **电子绘本朗读页（P4）** → 播放 / 翻页 / 可选 BGM → 结束或返回 → 书本写入书架。
- **识别为不含二维码的纸质绘本（Demo 模拟）**：**不经过 P3**，当前页切换为 **实时识别朗读页（P5）**（摄像头保持，逻辑见 §6.4）。

### 6.2 辅助流程：拍绘本

在 **P2** 切换「拍绘本」→ 用户点击屏幕触发识别 → 约 2s → **仅播放本段对应的本地预录音频**（**不在页面上展示识别文字或字幕**）→ 用户模拟翻页 → 再次识别则播放下一音频片段（间隔与 Demo 参数一致，如约 3s）。

### 6.3 音色克隆

首页 → 音色克隆 → 录音页 → 完成 → 成功页 → 保存 → 回首页；朗读时音色列表可见克隆项，**播放逻辑仍为预设 MP3**。

### 6.4 实时识别页（P5）与拍绘本的一致性

- **文案展示**：与 §6.2 一致 — **不展示假文案/识别结果文字**，仅保留摄像头画面、状态指示与音频播放。
- **入口**：**非独立 Tab**；唯一入口见 §6.1 — 从首页「开始读书」进入 **P2** 后，若在扫码场景下判定为 **不含二维码的纸质绘本（Demo 模拟）**，则由当前页过渡为 **P5**。

## 7. 逐页需求要点（实现检查表）

### P1 首页（书架）

- 顶部装饰：云、星、气球等 **CSS 动画**（避免过重 JS）。
- 标题：「我的绘本书架」。
- 主体：每行 2 本卡片；空状态小熊插画 + 文案。
- 底部 Tab：书架 / 开始读书 / 音色克隆。
- 逻辑：首次空；扫码成功后写入 LocalStorage；可提供 2–3 本内置假数据便于演示。

### P2 扫码 / 拍照页

- 关闭、模式切换、全屏预览、虚线框与提示。
- **扫码模式（含默认「开始读书」进入后的首屏）**：
  - 识别为 **带二维码的绘本（Demo 模拟）**：扫描态 → 2s → 震动（若支持）+ 本地 beep → 跳转 **P3**。
  - 识别为 **不含二维码的纸质绘本（Demo 模拟）**：**不进入 P3**，在当前预览流上切换为 **P5 实时识别朗读**（界面与状态机过渡由交互稿细化）。
- **拍绘本模式**：`simulateOCR` — 点击触发 → 约 2s → **仅播放对应本地预录音频**，**不展示任何识别文字**；**不跳转**其他路由页。
- **禁止**依赖外链识别服务。

### P3 识别结果页

- 烟花 / 书本飞入等入场动画（性能友好实现）。
- 展示书名、作者、「已加入书架」。
- 仅保留主按钮 **「马上开始读」**（**已取消「稍后再读」**）；返回首页依赖顶部关闭/返回或统一导航规则。
- 进入时自动写入书架。

### P4 电子绘本朗读页（横屏）

- 工具栏：返回、书名、BGM、更多。
- **内容区：仅展示当前页绘本图**（**无文字浮层、无逐字/逐句高亮**）。
- 底部：上一页 / 播放暂停 / 下一页 / 音色选择 / 页码。
- iOS **横屏 API 限制**：采用 **提示用户旋转设备 + CSS 横屏布局**，不依赖不可靠的系统锁屏 API。

### P5 实时识别朗读页

- 由 **P2** 在「无二维码绘本」场景转入；定时（如每 3s）模拟识别并 **循环播放本地音频**；识别过程可有光效、图标等 **非文案** 反馈。
- **不展示识别结果文字、字幕或假文案**（与 §6.2、§6.4 一致）。
- 与 §3.4 手势规则一致（若含拖拽翻页等）。

### P6 / P7 音色克隆

- MediaRecorder + 波形（AnalyserNode）；兼容失败时提示。
- 最短 5s、最长 15s；完成校验。
- 成功页：试听 Blob、角色预设名、自定义名称、写入 IndexedDB + LocalStorage 列表。

## 8. 数据与本地存储

| 数据 | 存储 | 说明 |
| ---- | ---- | ---- |
| 绘本列表与元数据 | LocalStorage | JSON，含内置 mock 与用户扫码写入项。 |
| 克隆音色 Blob | IndexedDB | 与 metadata 关联；列表摘要可存 LocalStorage。 |

不涉及任何服务端同步。

## 9. 资源与性能（摘要）

- 图片：**须支持 JPG、PNG**；在兼容与体积允许的前提下可另提供 **WebP** 作为优化，但不替代 JPG/PNG 需求。
- 音频：MP3，建议约 128kbps；BGM、beep、分页朗读分离存放 `public`。
- 动画：多用 `transform`，减少布局抖动。

---

## 10. 交互与反馈（全局）

- 页面转场、按钮按下、识别成功、加载态、波形等：按原「核心动画 / 反馈机制」章节执行，且遵守 §3.1。
- 震动：`navigator.vibrate` 可选调用（不支持则静默）。
- 权限拒绝：摄像头 / 麦克风统一友好引导，禁止跳转外链说明页。

## 11. 视觉规范（冻结方向）

- **主色**：阳光橙 `#FF9F3A`  
- **辅色**：天空蓝、草莓粉、柠檬黄、薄荷绿  
- **背景**：奶白 `#FFF8E7`  
- **主文字**：深炭灰 `#2C3E50`  
- **3D 卡通**：厚按钮、圆角卡片、高光与投影、派对氛围装饰。
